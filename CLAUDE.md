# Thunder Discussions

This project is a **non-code discussion hub** for [thunder-id/thunderid](https://github.com/thunder-id/thunderid).

## Purpose

- Brainstorm features, improvements, and architecture decisions for ThunderID
- Create and manage GitHub Issues, Discussions, Labels, Milestones, and Projects
- Track decisions, action items, and open questions across chat sessions
- Search and reference existing issues, PRs, and discussions

## Discussion Style

- **Do not just agree.** Challenge ideas with valid counter-arguments when available. If a point seems wrong or an important angle is missed, say so directly. This is a brainstorming space — rubber-stamping defeats the purpose.
- If everything genuinely looks solid, say so — but never skip the critical evaluation step.

## Rules

- **No code is written in this project.** All code changes happen in the thunderid repo itself.
- Every chat session should auto-update the project memory with key decisions, action items, and discussion outcomes before ending.
- Use `gh` CLI for all GitHub operations against `thunder-id/thunderid`.
- When creating issues or discussions, always check for duplicates first.
- Reference existing issues/discussions by number when relevant.

## GitHub Repo

- **Repo**: `thunder-id/thunderid`
- **Description**: ThunderID is a high-performance, open-source identity stack designed for developers to secure and manage access for humans, AI agents, and workloads through fully composable identity flows.

## Memory

This project uses the auto-memory system. Every chat session must persist:
1. Decisions made during the discussion
2. Action items created (with issue/discussion numbers)
3. Open questions that need follow-up
4. Any new context about the project direction or priorities

Memory lives in `.claude/projects/-Users-sahandilshan-IAM-thunder-developments-thunder-discussions/memory/` and is shared across all chat sessions in this project.
