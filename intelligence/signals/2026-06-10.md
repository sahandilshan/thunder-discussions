## Notable

- **[FIDO Alliance]** Announced formation of an Agentic Authentication Technical Working Group (April 2026) to develop three specs: Verifiable User Instructions (phishing-resistant agent authorization by users), Agent Authentication (verifying agents act within user-defined parameters), and Trusted Delegation for Commerce. OpenAI joined the FIDO Alliance board to contribute. Initial spec contributions from Google (AP2) and Mastercard (Verifiable Intent). — [FIDO Alliance announcement](https://fidoalliance.org/fido-alliance-to-develop-standards-for-trusted-ai-agent-interactions/) — capability: C2, C5, C10 — domain: IAM-for-AI

- **[WorkOS]** Released auth.md (May 2026): an open agent registration protocol built on OAuth RFC 9728 and ID-JAG. Apps publish a machine-readable `/.well-known/auth.md` file; agents discover it and register scoped OAuth credentials per-session instead of using static API keys. Not tied to WorkOS infrastructure. — [MarkTechPost](https://www.marktechpost.com/2026/05/25/workos-releases-auth-md-an-open-agent-registration-protocol-built-on-oauth-standards/) — capability: C1, C4, C10 — domain: IAM-for-AI

- **[HashiCorp Vault]** Vault Enterprise 1.21 (March 2026) added native SPIFFE authentication: agents holding an X509-SVID can now exchange it directly for a Vault token without a separately managed credential, closing the bootstrap-secret gap. Vault also issues X509-SVIDs to authenticated workloads so they can participate in the broader SPIFFE ecosystem. — [HashiCorp blog](https://www.hashicorp.com/en/blog/vault-enterprise-1-21-spiffe-auth-fips-140-3-level-1-compliance-granular-secret-recovery) / [InfoQ coverage](https://www.infoq.com/news/2026/03/hashicorp-vault-1-21/) — capability: C2, C6, C7 — domain: IAM-for-AI
