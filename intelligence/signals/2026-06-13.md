- **[Microsoft / SPIFFE]** **NEW (date unverifiable — surfaced now, may predate the 24h window).** Microsoft published **`microsoft/identity-spiffe`** — a public reference **prototype for governing agent-to-agent traffic** that combines **Entra Agent Identity** provisioning, **SPIFFE/SPIRE** workload SVIDs for workload-to-workload mTLS, and a **sidecar** enforcing mTLS + route-level RBAC + JWT validation + Conditional-Access-style risk signals before traffic reaches the app. Runs on Azure Container Apps with **optional GCP and GitHub Actions federation paths (no shared secrets)** and optional cross-cloud SPIFFE bundle federation. - **[IETF OAuth WG]** **BACKFILL / expansion of the cluster logged 06-10 (not fresh-dated June 12–13).** The AI-agent OAuth draft landscape is broader than the four drafts enumerated on 06-10. New-to-our-log entries: **`draft-mishra-oauth-agent-grants-01`** (Delegated Agent Authorization Protocol, DAAP; expires Sep 3 2026), **`draft-song-oauth-ai-agent-collaborate-authz-01`** (OAuth ext. for multi-AI-agent collaboration), **`draft-jia-oauth-scope-aggregation-00`** (scope aggregation for multi-step agent workflows — one authorization covering a whole workflow), **`draft-yao-agent-auth-considerations-00`**, and **`draft-hartman-credential-broker-4-agents-00`**. The **Agent Authorization Profile (AAP)** is now at **-01 (published Feb 7 2026)**, up from the -00 logged 06-10. **`draft-oauth-transaction-tokens-for-agents`** is now at **-06** (-05 dated Apr 4 2026), adding `actor`/`principal` context fields to propagate agent + initiating-principal identity through a trust domain. Signals a still-fragmented standards race, not a single converging draft. — [datatracker search](https://datatracker.ietf.org/doc/draft-mishra-oauth-agent-grants/) / [AAP](https://datatracker.ietf.org/doc/draft-aap-oauth-profile/) / [txn-tokens-for-agents](https://datatracker.ietf.org/doc/draft-oauth-transaction-tokens-for-agents/) / [scope-aggregation](https://datatracker.ietf.org/doc/draft-jia-oauth-scope-aggregation/) — capability: C3, C5, C7, C10 — domain: IAM-for-AI
