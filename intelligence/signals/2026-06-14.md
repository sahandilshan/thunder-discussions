## Notable
(Lower-priority but worth logging.)

- **[Microsoft / Windows + Entra]** **NEW-TO-OUR-LOG (Build 2026, dated June 2 2026 — backfill, ~12 days old, never logged here).** At Build 2026 Microsoft introduced **Microsoft Execution Containers (MXC)** — a policy-driven, **kernel-enforced** execution layer on Windows and WSL where developers declare what an agent can touch (files, network) and the Windows kernel enforces least-privilege containment, with cross-OS enforcement between Windows and Linux. Identity angle is the relevant part for us: **each agent gets its own Entra ID service principal with just-enough privileges instead of sharing the user's OAuth token**, Windows attributes all container activity to that identity, and (per Joy Chik) "the agent becomes a managed identity, no different from a server workload." Defender/Entra/Intune/Purview integration **in preview July 2026**. Distinct from the Agent 365 *shadow-AI discovery* facet logged 06-11 — this is OS-level agent identity + sandboxing, not detection. — [Windows Developer Blog (June 2 2026)](https://blogs.windows.com/windowsdeveloper/2026/06/02/windows-platform-security-for-ai-agents/) / [SC Media](https://www.scworld.com/brief/microsoft-introduces-execution-containers-for-ai-agents) / [Visual Studio Magazine](https://visualstudiomagazine.com/articles/2026/06/02/at-build-2026-microsoft-sets-up-windows-as-an-os-for-ai-agents.aspx) — capability: C1, C2, C3 — domain: IAM-for-AI
