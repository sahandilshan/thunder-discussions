- **[IETF / individual draft]** **NEW-TO-OUR-LOG (individual submission `draft-prakash-aip-00`; not in the OAuth agent-auth cluster characterized 06-13; arxiv companion dated March 2026 — backfill, never logged here).** **AIP (Agent Identity Protocol): Verifiable Delegation for AI Agent Systems** defines a cryptographic agent identity that flows *across protocol boundaries* — MCP tool calls and A2A agent-to-agent delegation. Core primitive is the **Invocation-Bound Capability Token (IBCT)**, fusing identity + attenuated authorization + provenance into an **append-only token chain** (compact mode = signed JWT for single-hop; chained mode = Biscuit token with Datalog policies for multi-hop), using **Ed25519** keys, *no blockchain/wallet*. Reference implementations in **Python + Rust** (cross-language interop); claimed overhead 0.22 ms over HTTP MCP. Distinct from the Microsoft `identity-spiffe` prototype (06-13) and the OAuth/AAP/txn-token drafts (06-13): this is a single cross-protocol delegation scheme spanning MCP *and* A2A, not an OAuth profile. Individual draft (-00), **not WG-adopted**. — [datatracker draft-prakash-aip](https://datatracker.ietf.org/doc/draft-prakash-aip/) / [IETF html archive](https://www.ietf.org/archive/id/draft-prakash-aip-00.html) / [arxiv 2603.24775](https://arxiv.org/abs/2603.24775) / [PyPI agent-identity-protocol 0.1.1](https://pypi.org/project/agent-identity-protocol/0.1.1/) — capability: C2, C3, C4, C5, C7, C10 — domain: IAM-for-AI
