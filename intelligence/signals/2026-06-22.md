## Notable

- **[Google / ARD spec]** **BACKFILL — published June 17, 2026; never logged here (a genuine miss, in-window across the Identiverse runs).** Google, **Microsoft, and Hugging Face** co-authored the **Agentic Resource Discovery (ARD)** specification — a standard for how websites/services advertise their agent-consumable resources (APIs, tools, data) to AI agents via an **`ai-catalog.json`** manifest, analogous to `robots.txt`/sitemaps for the agentic web. Backed by **Amazon, Cisco, GitHub, Salesforce, Snowflake, and Nvidia** (11 companies total). Sits at the **discovery layer beneath A2A/MCP** — how an agent finds *what* to call before auth/invocation — and is explicitly positioned by some coverage as a multi-vendor standard countering OpenAI/Anthropic ecosystem lock-in. Relevant to ThunderID's MCP-server-as-resource-server (#2973) surface: discovery is the step before the resource-server auth we track. Inference flagged: framing as an "OpenAI/Anthropic counter" is press characterization, not a stated goal. — [Google Developers Blog (announcement)](https://developers.googleblog.com/announcing-the-agentic-resource-discovery-specification/) / [Search Engine Journal](https://www.searchenginejournal.com/google-microsoft-back-draft-ai-agent-discovery-spec/579894/) / [Crypto Briefing (backers)](https://cryptobriefing.com/ard-ai-standard-google-microsoft-salesforce/) — capability: C4, C5, C10 — domain: IAM-for-AI

- **[Okta/Auth0 × Google Cloud]** **BACKFILL — June 16, 2026 (Identiverse week); never logged (distinct from the Okta XAA/Claude item logged 06-19 and the Ping "across AWS/Google Cloud/Cloudflare" item logged 06-16).** Okta and Google Cloud expanded their partnership to secure AI agents and the browser. Part one, **available now**: **Auth0 for AI Agents** integrated with the **Gemini Enterprise Agent Platform Runtime** — user authentication so only verified users activate an agent, plus a **Token Vault** that stores/manages/refreshes OAuth tokens so agents can act on a user's behalf against downstream/third-party services. Part two (**coming soon**): **Okta for AI Agents** tied to Gemini Enterprise for centralized visibility/policy control. Also ships Chrome Enterprise integrations (Universal Enrollment, Device Trust via Chrome Device Trust Connector, Apple Extensible SSO on macOS with Okta as IdP) targeting session-hijacking/credential theft. Tier-1 IdP wiring agent token-handling (C7) directly into a hyperscaler agent runtime — overlaps ThunderID's token-exchange / token-vaulting surface. — [Okta press release](https://www.okta.com/newsroom/press-releases/okta-teams-up-with-google-cloud-to-secure-the-ai-powered-workforce/) / [SiliconANGLE (June 16)](https://siliconangle.com/2026/06/16/okta-expands-google-cloud-partnership-secure-ai-agents-browser/) / [Techzine](https://www.techzine.eu/news/security/142204/okta-and-google-cloud-link-identity-to-ai-agents-and-browsers/) — capability: C2, C3, C7 — domain: IAM-for-AI

- **[C1 / C1 Autonomous Worker]** **NEW VENDOR (not in `TAXONOMY.md`). BACKFILL — June 15, 2026; never logged.** C1 (AI-native identity platform) launched **C1 Autonomous Worker (C1AW)**, billed as the first AI agent purpose-built for **enterprise identity governance work** — users ask it (via a Slack app) to revoke stale admin grants in Snowflake, build quarterly access reviews, pull audit evidence, or find deactivated users still listed as entitlement owners, and it completes the job by reasoning over data and acting **through C1's policy engine**. Governance model is the signal: **every C1AW action is bounded by the same policy engine that governs human users (it can only do what its operator may already do) and is attributed in C1's audit trail.** A clear **AI-for-IAM** entry (AI running IGA itself) — the thin column in our coverage — echoing SailPoint Agentic Acceleration (06-17) and Lumos Identity Agent Force (06-19). — [GlobeNewswire (June 15)](https://www.globenewswire.com/news-release/2026/06/15/3311817/0/en/C1-Launches-C1-Autonomous-Worker-a-Governed-AI-Agent-for-Enterprise-Identity-Work.html) / [Security Boulevard](https://securityboulevard.com/2026/06/c1-adds-autonomous-worker-to-automate-identity-governance-tasks/) / [VMblog](https://vmblog.com/news/c1-launches-c1-autonomous-worker-a-governed-ai-agent-for-enterprise-identity-work/) — capability: C8, C9 — domain: AI-for-IAM

- **[Google DeepMind / AI Control Roadmap]** **BACKFILL — published June 18, 2026; never logged (adjacent to identity-authz; flagged).** DeepMind published an **AI Control Roadmap** + companion guide ("Three Layers of Agent Security") that **discards the assumption of training-time alignment** and instead applies **enterprise insider-threat controls to deployed agents**: a **Supervisor Agent** for runtime monitoring of agent reasoning, **cryptographic signing of agent actions**, a **kill switch**, and a threat taxonomy mapped to **MITRE ATT&CK**, with Detection tiers D1–D4 and Prevention/Response tiers R1–R3 (lower-risk actions reviewed after the fact, higher-risk actions blocked in real time). More agent-containment/safety than identity-authz proper (flagged), but the "treat agents like privileged insiders — monitor, sign actions, hard-stop before irreversible" framing is the same architecture the runtime-authorization vendors (06-16→06-18) are converging on, expressed by a model lab. — [DeepMind blog](https://deepmind.google/blog/securing-the-future-of-ai-agents/) / [eWeek](https://www.eweek.com/news/deepmind-ai-agent-security-roadmap/) — capability: C3, C9 — domain: IAM-for-AI
