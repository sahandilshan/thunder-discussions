This run surfaced **four more that our logs never named**, all squarely in our lane (C1/C2/C3/C7):
  - **`draft-mishra-oauth-agent-grants-01` — Delegated Agent Authorization Protocol (DAAP).** Cryptographic agent identity via **DIDs**, human-consent OAuth-2.0-style grant flow, signed-JWT grant token with agent claims, online-verification revocation, **hash-chained append-only audit trail**, policy engine, and **multi-agent delegation with cascade revocation**. Rev 01 published **March 2, 2026** — **pre-window**. — [draft-mishra-oauth-agent-grants-01](https://datatracker.ietf.org/doc/draft-mishra-oauth-agent-grants/)
  - **`draft-mcguinness-oauth-actor-profile-00` — OAuth Actor Profile for Delegation.** For agents/workloads acting on behalf of users across org boundaries. Expires Nov 1, 2026 → published **~May 5, 2026** — **pre-window**. — [draft-mcguinness-oauth-actor-profile-00](https://datatracker.ietf.org/doc/html/draft-mcguinness-oauth-actor-profile-00)
  - **`draft-oauth-transaction-tokens-for-agents-06` — Transaction Tokens For Agents.** Extends OAuth Transaction Tokens with agent context: `act` field = the agent performing the action, `sub` = the principal (the agent itself when autonomous), so services in the call graph make granular access decisions. Rev 05 was **April 2026**; **rev 06's date is unverifiable** this run (datatracker 403). Possibly recent — flag for next run. — [draft-oauth-transaction-tokens-for-agents](https://datatracker.ietf.org/doc/draft-oauth-transaction-tokens-for-agents/)
  - **`draft-oauth-ai-agents-on-behalf-of-user-02` — OAuth 2.0 Extension: On-Behalf-Of User Authorization for AI Agents.** Rev 02; date unverifiable this run. — [draft-oauth-ai-agents-on-behalf-of-user](https://datatracker.ietf.org/doc/draft-oauth-ai-agents-on-behalf-of-user/)
