## High signal

- **[Anthropic]** **Workload Identity Federation (WIF) is now GA on the Claude Platform** — and our prior logs never captured it despite mining "Anthropic/MCP" daily. The **static `sk-ant-…` API key is now optional**: a workload presents a JWT from any OIDC-compliant IdP (**AWS IAM role, GCP/Kubernetes service account, Azure managed identity, GitHub Actions token, or Okta**), the Console maps the JWT claims (federation issuer + rule) to a **service account (`svac_…`)**, and Anthropic mints a **short-lived Claude access token** per request instead of a long-lived shared secret. Service accounts are first-class **non-human identities inside the Anthropic org** with their own roles + audit trail (which workload acted as which `svac_`). **Covers every Claude API endpoint, the SDKs, and Claude Code**; existing API keys keep working for incremental migration. **Dated June 17 — a backfill, not a June 28–29 drop** (flagged in Scout notes). Promoted to High: this is core Tier-2-vendor movement squarely on **C2 (agent authentication, OIDC-federated short-lived creds vs. static keys), C6 (workload identity federation), and C7 (short-lived/audience-scoped tokens)**, directly adjacent to ThunderID's token-exchange (RFC 8693) / short-lived-token / agent-as-NHI surface and worth a human-led cross-reference. — [Claude blog — WIF now GA](https://claude.com/blog/workload-identity-federation) / [Claude Platform docs — Workload Identity Federation](https://platform.claude.com/docs/en/manage-claude/workload-identity-federation) / [imisofts (June 19)](https://imisofts.com/blog/anthropic-workload-identity-federation-no-api-keys-news-june-19-2026/) / [Aembit analysis](https://aembit.io/blog/anthropic-workload-identity-federation-what-it-gets-right-and-what-it-still-doesnt-solve/) — capability: C2, C6, C7 — domain: IAM-for-AI
