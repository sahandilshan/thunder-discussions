- **[IETF / draft-diaconu-agents-authz-info-sharing]** Net-new to our logs and the most substantive fresh artifact this sweep: **`draft-diaconu-agents-authz-info-sharing-00`** — *Cross-Domain AuthZ Information sharing for Agents* (Jean Diaconu; v00 dated **Feb 6, 2026**, expires Aug 10, 2026; **Standards Track intent, individual / non-WG-adopted**). There is an **active OAuth-WG mailing-list thread** ("Thoughts on cross-domain authz info sharing for agents?"), so it has live list attention even if unadopted. Directly on ThunderID's cross-domain/federated agent-authorization + MCP surface (**C3** agent authorization, **C4** MCP support, **C10** standards participation); folds into the broad IETF agent-identity individual-draft cluster prior runs track (AAP/DAAP/actor-profile/actor-proofs/SPICE/AAuth/yao/klrc/song). **Logged Notable, not High:** net-new to logs but an **individual submission dated Feb 6** — out-of-window, not a July 7–8 WG movement. — [datatracker — draft-diaconu-agents-authz-info-sharing](https://datatracker.ietf.org/doc/draft-diaconu-agents-authz-info-sharing/) / [OAuth-WG list thread](http://www.mail-archive.com/oauth@ietf.org/msg25605.html) — capability: C3, C4, C10 — domain: IAM-for-AI
