## Notable
(Lower-priority but worth logging.)

- **[OpenID Foundation / AIIM CG]** **Net-new to our logs and the freshest on-taxonomy artifact this sweep:** the OpenID Foundation's **Artificial Intelligence Identity Management Community Group (AIIM CG)** issued a **Call for Participation for an MCP-security interoperability event** — *"Demonstrate MCP-based AI agent security with open identity standards"* — to be presented by **Atul Tulshibagwale (OIDF) and Erik Wahlström (Gartner)** at the **Gartner IAM Summit 2026 (Las Vegas)**. The interop explicitly tests the stack ThunderID is building: **OAuth 2.1 baseline authz, OAuth Client ID Metadata Document (CIMD) to identify clients by URL without prior registration, and Enterprise-Managed Authorization based on the Identity Assertion JWT Authorization Grant (ID-JAG)** for conveying identity across org boundaries; participants take a role (MCP client / OpenID Provider / OAuth AS / MCP gateway / MCP server) and must test ≥1 interaction with a complementary partner. Timeline: **weekly info calls from July 17**, **sign the Participation Agreement by Aug 4**, **≥1 successful partner test by Oct 16**. Directly on ThunderID's CIMD (#2975), MCP-client-as-OAuth-app (#2974), and token-exchange/ID-JAG surface (**C4** MCP support, **C10** standards participation, **C3** agent authz). **Logged Notable, not High:** it's a multi-vendor *event/coordination* signal (not a product or spec drop), and it maps onto CIMD/ID-JAG primitives already extensively tracked. **Dating flag:** exact publish date unconfirmed (`openid.net` returned HTTP 403 this run); the "calls from July 17 / commit by Aug 4" timeline places the post in early–mid July 2026 (i.e. near-window, likely this week). — [OpenID — Call for participation: Demonstrate MCP-based AI agent security](https://openid.net/call-for-participation-demonstrate-mcp-based-ai-agent-security-with-open-identity-standards/) — capability: C4, C10, C3 — domain: IAM-for-AI

- **[OpenID Foundation / AuthZEN WG]** **Net-new to our logs (backfill):** the **AuthZEN Working Group approved two official Working Group Drafts** for the agent era — **AARP (Access Request and Approval Profile)**, which handles the "policy can't authorize *yet* because a prerequisite must first be satisfied" case (request/approval as a first-class authz outcome — relevant to human-in-the-loop agent approvals), and critically **COAZ — the *AuthZEN Profile for Model Context Protocol Tool Authorization*** — which standardizes mapping MCP tool-call information into the AuthZEN Subject-Action-Resource-Context (SARC) model so an **API/AI gateway, service mesh, or MCP server can externalize MCP-tool authorization to an AuthZEN-compatible PDP**. Directly on ThunderID's MCP + externalized-authz surface (**C4** MCP support, **C3** agent authz, **C10** standards). **Logged Notable with an out-of-window flag:** the announcement frames itself around "the identity community gathers at **Identiverse 2026**" (early June), so this is an **~June, out-of-window** WG-draft approval, surfaced here because it was **never logged in prior runs** and COAZ (AuthZEN-for-MCP-tools) is squarely deep-dive-worthy — not because it is July 10–11 movement. Ties to the already-tracked Keycloak-26.7.0 AuthZEN-PDP and the AIIM MCP-interop item above. — [OpenID — advances authorization for the agent era with new AuthZEN Working Group Drafts](https://openid.net/openid-foundation-advances-authorization-for-the-agent-era-with-new-authzen-working-group-drafts/) — capability: C4, C3, C10 — domain: IAM-for-AI

- **[Keycloak]** **Facet of the already-logged 26.7.0 GA (July 9), not previously called out:** 26.7.0 adds **identity providers federating *client* authentication** — clients can now authenticate to Keycloak with **SPIFFE JWT-SVIDs, Kubernetes service-account tokens, or tokens issued by an external OIDC IdP** (explicitly positioned for integration with **SPIRE / agent-identity systems**). This is a **workload-identity-federation** hook (**C6** workload identity, **C2** agent authentication) that the 07-10 log's 26.7.0 write-up (which covered CIMD/MCP, AuthZEN-PDP, ID-JAG, SSF, SCIM, HA, OID4VCI) did **not** enumerate. **Not net-new movement** — same July-9 release already logged High on 07-10 — logged here only as a newly-surfaced facet worth a human-led note against ThunderID's workload/SPIFFE surface. — [Keycloak blog — 26.7.0 released](https://www.keycloak.org/2026/07/keycloak-2670-released) / [GitHub — keycloak 26.7.0](https://github.com/keycloak/keycloak/releases/tag/26.7.0) — capability: C6, C2 — domain: IAM-for-AI
