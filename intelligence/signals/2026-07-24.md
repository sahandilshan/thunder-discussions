# Signals — 2026-07-24

## High signal

- **[IETF / DAWN BoF]** **NET-NEW-TO-LOGS, in-window.** IETF 126 Vienna ran a **second WG-forming agent BoF besides agentproto: `DAWN` — "Discovery of Agents, Workloads, and Named Entities"** (met **Tue July 21**), seeking a Working Group to standardize an **automated, decentralized, interoperable discovery mechanism** for agents that have *no pre-configured knowledge of each other*, explicitly scoped to **agent-to-agent discovery, cross-org credential/task delegation, and privacy-preserving user-context passing**. Prior scout runs flagged only agentproto as "the" agent BoF — there were in fact **≥3 agent-focused BoFs this week** (agentproto + DAWN WG-forming, DMSC non-WG-forming). DAWN is the discovery/naming layer directly upstream of agent identity. Anchor drafts: **`draft-farrel-dawn-terminology-03`** (DAWN terminology, Adrian Farrel) and **`draft-jimenez-dawn-discovery-landscape-00`** (survey of AI-agent discovery mechanisms). — [DAWN terminology draft](https://datatracker.ietf.org/doc/draft-farrel-dawn-terminology/) · [discovery-landscape survey](https://datatracker.ietf.org/doc/html/draft-jimenez-dawn-discovery-landscape-00) · [IETF 126 new-topics](https://www.ietf.org/blog/ietf126-new-topics/) — capability: C1, C5, C10 — domain: IAM-for-AI

- **[IETF / DMSC BoF]** **NET-NEW-TO-LOGS, in-window.** Third agent BoF at IETF 126: **`DMSC` — "Dynamic Multi-Agent Secured Collaboration"** (non-WG-forming; profiled the landscape and gaps to feed agentproto). Proposed scope is squarely IAM-for-AI: **agent registration/classification, peer discovery, request routing, secure inter-agent comms, interoperable agent identity binding, authentication/authorization, and verifiable trust anchors for cross-domain collaboration** — with **network-visible agent identifiers** that represent an agent independent of host/location, and per-agent capability entries synced across domains with attached security attributes. Anchor draft: **`draft-li-dmsc-inf-architecture-06`** + BoF request `bofreq-wang-dynamic-multi-agents-secured-collaborationdmsc`. Overlaps ThunderID's agent-identity + cross-domain delegation surface directly. — [DMSC architecture draft](https://datatracker.ietf.org/doc/html/draft-li-dmsc-inf-architecture-06) · [DMSC BoF request](https://datatracker.ietf.org/doc/bofreq-wang-dynamic-multi-agents-secured-collaborationdmsc/) · [DMSC charter (GitHub)](https://github.com/ietf-dmsc/Charter/blob/main/dmsc-charter.md) — capability: C1, C3, C5, C10 — domain: IAM-for-AI

## Notable

- **[Innovation Labs / DNSid]** **NET-NEW-TO-LOGS** (announced **July 15**, out-of-window; DNSid the project name appeared in prior "no-movement" lists but the Cerf/IETF-draft development was never logged). **Internet pioneer Vint Cerf joined the Advisory Council of Innovation Labs** (a division of registry operator **Identity Digital**) to advance **DNSid** — a proposed open standard giving each AI agent a **durable identifier tied to an existing DNS domain name with cryptographic proofs**, framed as a **"license plate / passport" for accountability** (who stands behind an agent), *not* a trust score. Innovation Labs has **submitted an Internet-Draft to the IETF**. A registry-operator + Cerf entry into agent identity is a notable new axis distinct from the OAuth/MCP token track.
